## 第一部分：专业化、学术化描述版本

### 1. 研究背景与问题定义

本研究聚焦于超声脑部外侧裂 A3 切面的半监督分割任务。与常规器官分割相比，该任务具有三个显著挑战：

1. 标注代价高，导致有标签样本数量有限，半监督学习成为必要路径。
2. 外侧裂在 A3 切面中通常呈细长、低占比、边界弱对比的结构，容易发生断裂、粘连或空预测。
3. 超声图像存在散斑噪声、声影伪影和跨病例成像差异，使得 foundation model 在该场景中的稳定性并非天然成立。

在此背景下，KnowSAM 提供了一个有代表性的 baseline：它结合 SGDL 双分支分割网络和 SAM 的提示式分割能力，通过 SAM 辅助蒸馏提升半监督分割效果。然而，针对当前 A3 外侧裂任务，KnowSAM 仍然存在结构性瓶颈，而不仅仅是损失函数设计不充分。

### 2. Baseline 的核心瓶颈分析

KnowSAM 的原始信息流可以概括为：

```text
输入图像
  -> SGDL 产生 UNet / VNet / fusion_map
  -> fusion_map 直接作为 SAM 的 mask prompt
  -> SAM 输出分割结果
  -> SAM 结果以等权知识蒸馏方式监督学生分支
```

该 baseline 的问题主要体现在以下三个方面：

1. `fusion_map` 被直接作为 prompt 输入 SAM，但在半监督场景中，`fusion_map` 本身可能携带伪标签噪声，因此 prompt 质量不稳定。
2. SAM 被默认视为可靠教师，但在超声低对比边界区域，SAM 输出并非始终可信，等权蒸馏会放大错误监督。
3. 无标签样本与伪标签在训练中被近似等价使用，没有结合 A3 外侧裂的结构特性进行质量调度。

因此，本工作不将创新定义为“叠加几个新模块”，而是将其定义为：针对 KnowSAM 核心信息流进行框架级重构，使提示生成、教师监督与伪标签利用形成统一的可靠性闭环。

### 3. 总体框架：A3-RCP-KnowSAM

我们提出的框架可命名为：

**A3-RCP-KnowSAM**

全称为：

**Anatomy-aware Reliability-Calibrated Prompt Co-evolution KnowSAM**

其核心思想是：不再将 SAM 简单作为被动的外部教师，而是将其纳入一个由 SGDL 与 SAM 共同参与、并由可靠性校准驱动的协同框架中。

新框架包含三项相互耦合的创新：

1. RCP：Reliability-Calibrated Consensus Prompt  
   用多分支共识替代原始 noisy fusion prompt。
2. BRCE：Bidirectional Reliability-guided Co-evolution  
   将 SGDL 到 SAM 的提示质量和 SAM 到 SGDL 的教师质量统一到一个可靠性闭环中。
3. AQC：Anatomical Quality Controller  
   将 A3 外侧裂的结构先验引入无标签样本调度，使伪标签学习具有解剖学约束。

### 4. 创新点一：RCP，可靠性校准的共识提示机制

#### 4.1 动机

在 baseline 中，SAM 的 mask prompt 直接来自 `fusion_map`。这一设计隐含了一个较强假设，即融合分支的输出已经足够稳定。但对于 A3 外侧裂超声图像而言，这一假设并不成立，尤其在边界不清晰或前景面积极小的样本中，fusion_map 往往容易出现局部误激活。

如果这类不稳定的 fusion_map 被直接送入 SAM，则错误提示会进一步扰动 SAM 输出，导致后续蒸馏过程持续接收偏差教师信号。

#### 4.2 方法

我们不再直接使用单一 fusion 输出，而是先由 UNet、VNet 和 fusion head 的概率预测构建三分支共识，再对共识进行可靠性校准。

其形式可表示为：

```text
P_cons = Sharpen((P_unet + P_vnet + P_fusion) / 3)
```

再利用共识熵与双分支前景差异构造 prompt 可靠性：

```text
U_prompt(x) = [H(P_cons(x)) + |P_unet_fg(x) - P_vnet_fg(x)|] / 2
R_prompt(x) = clamp(exp(-alpha * U_prompt(x)), r_min, 1)
```

最终 prompt 不再是硬性依赖某一分支结果，而是在高可靠区域保留共识分布，在低可靠区域退回均匀先验，从而避免错误 prompt 对 SAM 的强驱动。

#### 4.3 学术贡献

RCP 的创新不在于简单更换 prompt 形式，而在于将 KnowSAM 从“单分支直接提示”升级为“多分支共识校准提示”。这属于提示生成机制层面的框架优化，而非单纯的附加损失项。

### 5. 创新点二：BRCE，可靠性闭环协同进化

#### 5.1 动机

在原始 KnowSAM 中，SAM 监督学生网络的方式是静态的、单向的。SAM 先被 prompt 驱动生成预测，再以统一权重蒸馏给 UNet 和 VNet。这种设计忽略了两个关键问题：

1. prompt 质量会直接影响 SAM 输出质量；
2. SAM 输出质量本身也应被动态评估，而不是默认等权可信。

#### 5.2 方法

BRCE 将可靠性建模前后打通，形成闭环：

```text
SGDL 三分支输出
  -> 构造可靠性校准 prompt
  -> SAM 生成预测
  -> 基于 SAM 熵、fusion 熵、双分支分歧计算 teacher reliability
  -> 以 reliability-weighted KD 监督 UNet / VNet
```

teacher reliability 的构造形式为：

```text
U_teacher(x) =
  [H(P_sam(x)) + H(P_fusion(x)) + |P_unet_fg(x) - P_vnet_fg(x)|] / 3

R_teacher(x) = clamp(exp(-beta * U_teacher(x)), t_min, 1)
```

蒸馏不再是等权 KL，而是：

```text
L_KD =
  sum_x R_teacher(x) * KL(P_sam(x) || P_student(x))
  / sum_x R_teacher(x)
```

#### 5.3 学术贡献

BRCE 的贡献在于把 KnowSAM 的 SAM 分支从“静态教师”改造成“可靠性可校准的协同参与者”。也就是说，SAM 不再只是输出一个结果给学生网络，而是在 prompt 阶段和 teacher 阶段都被纳入一个统一的可信度建模体系中。

这使得框架具有更强的 foundation model-task model 协同优化属性，也更符合当前高水平论文对于“信息流重构”和“统一训练机制”的要求。

### 6. 创新点三：AQC，A3 解剖质量控制器

#### 6.1 动机

半监督学习的关键并不只是“利用更多无标签样本”，而是“有选择地利用无标签样本”。在 A3 外侧裂场景中，这一点尤其重要，因为外侧裂本身面积小、形态细长、边界模糊，如果伪标签质量不佳，则很容易引导模型学习到空前景或异常膨胀的分割模式。

#### 6.2 方法

AQC 将样本质量定义为三类因素的联合结果：

1. prompt 阶段的可靠性；
2. teacher 阶段的可靠性；
3. A3 外侧裂的解剖结构合理性。

具体而言，我们通过前景面积比例构造一个宽松的软解剖先验，并与可靠性评分共同决定样本权重：

```text
Q_i = f(R_prompt_i, R_teacher_i, AnatomyScore_i)
```

随后在 MixUp 和无标签伪监督阶段，以质量加权形式进行 CE + Dice 训练。

同时，我们引入两类结构控制项：

1. 边界控制：通过 Sobel 梯度强化边界学习；
2. 软面积先验：避免空预测和大面积误分割。

#### 6.3 学术贡献

AQC 的意义在于，它不是简单“过滤伪标签”，而是将解剖结构知识嵌入半监督学习调度器中，使无标签监督不再是经验性的，而是具有明确任务先验支撑的。

对于 A3 外侧裂这种细结构目标，这类“anatomy-aware scheduling”比一般的困难样本挖掘更具任务针对性。

## 第二部分：相对白话、适合组会分享的描述版本

### 1. 可以怎么开场

这次我想汇报的重点，不是“我在 KnowSAM 上多加了几个 loss”，而是我觉得原来的 baseline 在信息流上就有问题，所以我想做的是框架层面的改造。

简单来说，KnowSAM 原来是这样干的：

1. 先用 SGDL 分出一个 `fusion_map`；
2. 再把这个 `fusion_map` 直接拿去提示 SAM；
3. 然后再让 SAM 反过来教 UNet 和 VNet。

问题在于，如果一开始这个 `fusion_map` 就不太靠谱，那后面 SAM 其实也是被错误提示带偏的。这样一来，SAM 再反过来蒸馏学生网络，整个链条就容易把错误不断放大。

所以我现在的思路是：不是继续往后面补模块，而是先把这条链条改掉。

### 2. 我的核心想法是什么

一句话概括：

**先想办法让 SAM 接收到更靠谱的提示，再让 SAM 只在自己靠谱的时候去教学生网络，最后再决定哪些无标签样本值得重点学。**

也就是说，我把整个过程拆成了三个连续的问题：

1. 怎么给 SAM 一个更稳的提示？
2. 怎么判断 SAM 什么时候真的可以当老师？
3. 怎么判断哪些伪标签样本值得重点利用？

### 3. 第一部分：先把 prompt 做稳

以前 baseline 是直接用 `fusion_map` 提示 SAM。  
我现在不这么做了，而是把 UNet、VNet 和 fusion 三个结果先做一个“共识”，看看它们是不是基本一致。

如果三个分支比较一致，那我就相信这个 prompt 多一点。  
如果它们差异很大，或者本身熵很高，那我就说明这个位置不太确定，这时候就不要强行把一个不靠谱的 mask 塞给 SAM。

所以这里的本质不是“换了个 prompt”，而是做了一个**可靠性校准的 prompt 生成机制**。

你可以把它理解成：  
以前是“一个人说了算”，  
现在是“三个人先商量一下，再决定怎么提示 SAM”。

### 4. 第二部分：SAM 不是默认可靠老师

我觉得 baseline 里还有一个问题，就是它默认认为 SAM 输出出来的结果都值得蒸馏。

但在超声图像里，尤其是外侧裂这种边界不清楚的小结构，SAM 有时候也会不稳定。  
所以我现在不是让 SAM 无条件教学生，而是先判断它在每个位置到底有多靠谱。

这个判断主要看三件事：

1. SAM 自己的预测是不是高熵；
2. fusion 结果是不是也不稳定；
3. UNet 和 VNet 在这个位置是不是互相打架。

如果这几个信号都说明这里不稳定，那这个地方的蒸馏权重就降低。  
如果这里比较稳定，那蒸馏就加强。

这样改完以后，SAM 就不是一个“默认永远正确”的老师，而是一个“只有在自己比较确定的时候，才更有发言权”的老师。

### 5. 第三部分：无标签样本不能一视同仁

第三个问题是伪标签。  
以前做半监督的时候，很多方法默认只要有伪标签就拿来训练，但我觉得在 A3 外侧裂这个任务上，这样很容易把坏样本也学进去。

因为外侧裂本来就是一个又细又小的结构，如果某个伪标签看起来特别大、特别厚，或者干脆没分出来，那其实它就不应该和高质量样本一样参与训练。

所以我这里做了一个样本质量控制器，大概就是判断：

1. 这个样本在 prompt 阶段靠不靠谱；
2. 它在 teacher 阶段靠不靠谱；
3. 它分出来的前景面积是不是大致符合外侧裂的结构特征。

如果这三方面都还可以，那这个无标签样本就给更高权重。  
如果明显不行，那也不是完全丢掉，而是降低它对训练的影响。

这一步我觉得很重要，因为它把“怎么用无标签数据”这件事，从经验调参变成了任务驱动的质量调度。

### 6. 为什么我觉得这比“模块拼接”更像一个创新框架

因为这三部分不是彼此独立的。

它们实际上构成了一条新的闭环：

1. 更可靠的 SGDL 共识，去生成更可靠的 prompt；
2. 更可靠的 prompt，去得到更稳定的 SAM 输出；
3. 更稳定的 SAM 输出，再以加权方式监督学生网络；
4. 同时这些可靠性信息又用来决定伪标签样本该怎么学。

所以它不是：

```text
baseline + loss1 + loss2 + loss3
```

而更像是：

```text
把 baseline 的核心交互链条重构了
```

这一点我认为更接近顶会想看到的“统一框架创新”，而不是简单的工程叠加。

### 7. 我现在已经做到哪一步了

目前不是停留在想法层面，而是已经做了实现和本地跑通：

1. 新的 prompt 机制已经接进训练代码；
2. 可靠性加权蒸馏已经实现；
3. 伪标签质量加权和 MixUp 分支也已经接通；
4. 本地用 conda 的 `pytorch` 环境在 GTX 1650 上完成了 smoke test；
5. 训练、验证、best model 保存和 MixUp 分支都已跑通。

也就是说，现在这个方案已经不是“提案”，而是“有代码、有日志、能继续上服务器跑完整实验”的状态。

### 8. 后续怎么推进

接下来最重要的不是继续加新东西，而是把实验做扎实：

1. 先和原始 KnowSAM 做完整 baseline 对比；
2. 再做三个模块的消融；
3. 再看 reliability 曲线和结构指标有没有体现出预期趋势；
4. 如果结果支持，就可以进一步整理成论文问题定义和方法框架图。

### 9. 组会上可以这样收束

最后我想强调的是，这项工作的重点不是“把 SAM 用进半监督分割”，因为别人已经做过很多类似事情了。

我这里真正想解决的是：

**在超声这种高噪声、小样本、弱边界的场景里，SAM 到底应该怎么被可靠地接入 baseline。**

所以我的方案本质上是在回答三个问题：

1. 怎么给 SAM 一个靠谱的提示？
2. 怎么让 SAM 在靠谱的时候再去教学生？
3. 怎么让无标签样本按质量被利用，而不是一股脑全学进去？

如果这三件事能够被实验验证，那么这个工作就不仅仅是一个 baseline 改进，而是一个比较完整的、面向 A3 超声外侧裂半监督分割的框架设计。

## 第三部分：可直接口头使用的简短总结

### 30 秒版

我这次不是在 KnowSAM 上简单加几个 loss，而是针对它的核心信息流做了框架优化。原来的问题是 noisy fusion prompt 会带偏 SAM，而 SAM 又会把不稳定的结果等权蒸馏回学生网络。我现在做的是一个可靠性校准闭环：先用多分支共识生成更稳的 prompt，再按 SAM 和分支的一致性决定蒸馏强度，最后结合 A3 外侧裂的结构先验去调度伪标签学习。

### 1 分钟版

我这项工作针对的是 A3 外侧裂半监督分割。KnowSAM 作为 baseline 的优点是把 SGDL 和 SAM 结合起来了，但问题是它默认 fusion_map 可以直接提示 SAM，也默认 SAM 输出始终适合作为教师。我现在做的改动是把这两个默认假设去掉：第一，用 UNet、VNet 和 fusion 的共识生成一个可靠性校准 prompt；第二，对 SAM 输出做像素级可靠性评估，只在高可信区域强化蒸馏；第三，再把这些可靠性信息和外侧裂的面积、边界先验结合起来，控制无标签样本的利用强度。这样整个框架就从单向提示、单向蒸馏，变成了可靠性驱动的闭环协同。

### 2 分钟版

我的工作核心不是继续往 KnowSAM 上补模块，而是重构它的交互逻辑。原始 KnowSAM 用 fusion_map 直接提示 SAM，再让 SAM 反过来监督 UNet 和 VNet。但对于 A3 超声外侧裂这种小目标、弱边界场景，这条链路的问题是：一开始的 prompt 可能就不可靠，后面的蒸馏也会被连带污染。基于这个观察，我提出了一个 A3-RCP-KnowSAM 框架。第一步，用三分支共识来替代单一 fusion prompt，并对 prompt 做可靠性校准；第二步，不再把 SAM 当成静态教师，而是根据 SAM 熵、fusion 熵和双分支分歧对蒸馏做加权；第三步，引入 A3 解剖质量控制器，根据可靠性和结构合理性去调度伪标签训练。这样，prompt 质量、teacher 质量和伪标签质量就被统一到了一个闭环框架里。当前代码已经实现并完成了本地 smoke test，下一步就是上服务器做完整消融和对比实验。
